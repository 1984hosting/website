Note:

{{#context.packaged}}

Certbot is packaged in EPEL (Extra Packages for Enterprise Linux). To use Certbot, you must first enable the EPEL repository.

{{/context.packaged}} {{^context.packaged}}

Not all of Certbot's dependencies are available in the standard repositories. To use Certbot, you must first enable the EPEL (Extra Packages for Enterprise Linux) repository.

{{/context.packaged}} {{#context.rhel}}

This can be done by downloading the proper epel rpm and installing it: 

        $ rpm -Uvh https://dl.fedoraproject.org/pub/epel/epel-release-latest-{{context.version}}.noarch.rpm
Once you've done this, you can verify that it worked by running: 
        $ yum repolist | grep epel

{{#context.packaged}}

You must also enable the optional channels, which can be done by running these commands: 

          $ subscription-manager repos --enable rhel-7-server-extras-rpms
          $ subscription-manager repos --enable rhel-7-server-optional-rpms

This can be done by running the command: 

          $ yum install epel-release

Additionally if you are using RHEL on EC2, you can enable the optional channel by running: 

        $ yum -y install yum-utils
        $ yum-config-manager --enable rhui-REGION-rhel-server-extras rhui-REGION-rhel-server-optional
{{/context.packaged}} {{/context.rhel}} {{#context.centos}}

This can be done by running the command: 

        $ yum install epel-release

{{/context.centos}}
